## 1. OpenAI한테 HTTP로 직접 질문하기

평소에는 `openai` 라이브러리를 써서 질문했는데,  
오늘은 그냥 **HTTP 요청(requests)으로 직접** OpenAI 서버에 질문을 보내봤다.

결과는 똑같다. 단지 더 원시적인 방법으로 한 것.

> 비유: 배달앱 쓰는 것(라이브러리) vs 식당에 직접 전화하는 것(HTTP 직접 요청)

## 2. 대화를 이어가기 — previous_response_id

AI한테 이런 질문을 해봤다:

```
1번 질문: "대한민국의 수도는 어디야?"
2번 질문: "그 도시의 인구는 몇이야?"
3번 질문: "그 도시에서 가볼만한 곳 세 곳만 추천해줘"
```

근데 2번, 3번 질문에서 "그 도시"가 어디인지 AI가 어떻게 알까?

**비밀:** 질문할 때 "이전 답변 ID"를 같이 보내면 AI가 맥락을 기억한다.

```
1번 질문 → AI가 답변 + ID(번호표) 돌려줌
2번 질문 + 1번 ID → AI가 "아, 서울 얘기구나" 하고 이어서 답변
3번 질문 + 2번 ID → 또 이어서 답변
```

ID를 안 보내면? AI가 "그 도시가 어디요?" 하고 모른다.

## 3. WebSocket — 실시간 채팅

### HTTP랑 뭐가 다른가?

| | HTTP | WebSocket |
|--|------|-----------|
| 비유 | 편지 주고받기 | 전화 통화 |
| 연결 | 물어볼 때마다 새로 연결 | 한 번 연결하면 계속 유지 |
| 누가 먼저? | 클라이언트만 먼저 말할 수 있음 | 서버도 먼저 말할 수 있음 |
| 쓰는 곳 | 일반 웹페이지 | 채팅, 주식, 게임 |

### 오늘 만들어본 것들

**1단계 - 기본 WebSocket 서버**
- 파이썬 서버 켜놓으면, 브라우저에서 접속해서 메시지를 주고받을 수 있음
- 브라우저에서 보낸 메시지를 서버가 그대로 돌려줌 (에코)

**2단계 - Flask에 WebSocket 붙이기**
- 원래 Flask는 WebSocket을 못 함
- `flask-sock` 라이브러리 설치하면 Flask에서도 WebSocket 가능
- `@app.route` 대신 `@sock.route` 쓰면 됨

**3단계 - Socket.IO**
- `flask-socketio` 라이브러리
- 가장 편리한 방법
- `broadcast=True` 옵션 하나로 **접속한 모든 사람한테 동시에** 메시지 보낼 수 있음
- → 이게 있어야 진짜 채팅방이 됨 (내가 보낸 거 다른 사람도 봐야 하니까)

## 4. yield — 값을 조금씩 꺼내기

SSE랑 스트리밍을 이해하려면 이걸 먼저 알아야 함.

### return이랑 뭐가 다른가?

**return** - 값 하나 주고 함수 완전 종료

```
함수 호출 → 값 받음 → 끝. 다시 부르면 처음부터 시작.
```

**yield** - 값 하나 주고 잠깐 멈춤 (함수가 살아있음)

```
함수 호출 → 값 1 받음 → 잠깐 멈춤
다시 요청 → 값 2 받음 → 잠깐 멈춤
다시 요청 → 값 3 받음 → 잠깐 멈춤
더 없으면 → 종료
```

> 비유: return은 게임을 끄면 처음부터, yield는 세이브하고 끄면 거기서 이어하기

### 왜 필요한가?

100만 개 숫자를 다 메모리에 올리는 것보다,  
필요할 때 하나씩 꺼내면 훨씬 효율적.

> 비유: 책 100권을 다 외우는 것 vs 필요한 페이지만 펼쳐보는 것

## 5. SSE (Server-Sent Events) — 서버가 먼저 데이터 보내기

### 개념

보통 웹은 **클라이언트가 먼저 요청**해야 서버가 답한다.  
SSE는 반대로 **서버가 알아서 데이터를 밀어준다.**

```
[일반 HTTP]  클라이언트: "줘!" → 서버: "여기!" → 끝

[SSE]        클라이언트: "구독할게요~"
             서버: "알겠어, 새 데이터 생길 때마다 보내줄게"
                  → 계속 보내줌
```

> 비유: 일반 HTTP = 편의점 가서 물 사오기 / SSE = 정수기 구독해서 알아서 배달옴

### 어떻게 만드나?

서버에서 **yield로 응답을 끊지 않고** 계속 흘려보내면 된다.  
데이터는 `data: 내용` 형식으로 보내야 하는 게 웹 표준 규칙.

브라우저에서는 `EventSource`로 구독하면 메시지 올 때마다 자동으로 실행됨.

### SSE vs WebSocket 언제 쓰나?

| 상황 | 쓸 것 |
|------|-------|
| 서버→클라이언트로만 보내면 됨 (알림, 뉴스피드, AI 답변) | SSE |
| 서로 주고받아야 함 (채팅, 게임) | WebSocket |

## 6. OpenAI 스트리밍 — AI가 타이핑하듯 답변하기

### 개념

```
일반 방식: AI가 답변 다 만들고 → 한 번에 짠! 하고 보냄
스트리밍: AI가 "안" 만들면 바로 보냄 → "녕" 만들면 바로 보냄 → "하" ...
```

> 비유: 유튜브에서 영상 다 받고 보는 것 vs 받으면서 바로 보는 것

### 어떻게 만드나?

**서버에서:** OpenAI에 요청할 때 `stream=True` 옵션 하나만 추가  
그러면 글자 조각이 올 때마다 yield로 바로 브라우저에 전달

**브라우저에서:** 조각이 올 때마다 화면에 붙여서 보여줌  
→ 글자가 타이핑되듯 하나씩 나타나는 효과

### 2가지 버전 만들어봤음

| 버전 | 차이점 |
|------|--------|
| `1.content/` | SSE 형식으로 보냄 (`data: {...}`) — 더 정석 방법 |
| `2.simple/` | 그냥 텍스트로 바로 보냄 — 더 단순한 방법 |
