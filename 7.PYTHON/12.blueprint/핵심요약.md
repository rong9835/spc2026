# Flask Blueprint 쉬운 설명

## 먼저 Flask가 뭔지부터

Flask = 파이썬으로 웹사이트 만드는 도구

웹사이트에는 주소가 있음

```
naver.com/           ← 메인 페이지
naver.com/news       ← 뉴스 페이지
naver.com/sports     ← 스포츠 페이지
```

Flask에서는 이 주소들을 이렇게 만듦

```python
@app.route('/')
@app.route('/news')
@app.route('/sports')
```

---

## 문제 발생

주소가 많아지면 `app.py` 파일 하나에 다 넣게 되는데...

```python
# app.py 가 엄청 길어짐 😱
@app.route('/')
@app.route('/news')
@app.route('/news/sports')
@app.route('/news/politics')
@app.route('/user')
@app.route('/user/login')
@app.route('/user/signup')
@app.route('/admin')
@app.route('/admin/dashboard')
# ... 100개 계속
```

파일이 너무 길어져서 뭐가 어딨는지 못 찾음

---

## Blueprint = 파일 나누기

파일을 기능별로 쪼개는 것

```
app.py           ← 메인 (짧고 깔끔하게)
user_routes.py   ← 유저 관련 주소만 모아둠
admin_routes.py  ← 관리자 관련 주소만 모아둠
```

그게 전부임. 어려운 개념 아니고 그냥 **코드 정리 방법**

---

## 사용법 3단계

### 1단계 - 분리된 파일에서 Blueprint 만들기

```python
# user_routes.py 파일
from flask import Blueprint, render_template

user_blueprint = Blueprint('user', __name__)
# 그냥 "나는 user 블루프린트야" 라고 선언하는 것

@user_blueprint.route('/')   # app. 대신 user_blueprint. 로 바꾼 것뿐
def user_page():
    return render_template('user.html')
```

원래 `@app.route` 쓰던 자리에 `@user_blueprint.route` 쓰는 것만 다름

---

### 2단계 - app.py 에서 "이 파일도 써줘" 하고 등록

```python
# app.py
from flask import Flask
from user_routes import user_blueprint    # user_routes.py 파일에서 가져옴
from admin_routes import admin_blueprint

app = Flask(__name__)

app.register_blueprint(user_blueprint, url_prefix="/user")
#                                       주소 앞에 /user 를 붙여줌
app.register_blueprint(admin_blueprint, url_prefix="/admin")
```

---

### 3단계 - 결과

| 파일 | 주소 |
|------|------|
| app.py | `localhost/` |
| user_routes.py | `localhost/user/` |
| admin_routes.py | `localhost/admin/` |

---

## HTML 에서 링크 연결할 때

```html
<a href="{{ url_for('home') }}">홈</a>
<a href="{{ url_for('user.user_page') }}">유저</a>
<!--              블루프린트이름.함수이름  -->
```

블루프린트 안에 있는 함수는 앞에 `블루프린트이름.` 을 붙여줘야 함

---

## 2.more 폴더는 뭐가 다름?

기능이 3개(유저, 관리자, 상품)로 늘어나서 **폴더로도 나눈 것**

```
1.basic 구조              2.more 구조
────────────────          ────────────────
app.py                    app.py
user_routes.py            routes/
admin_routes.py             user/user_routes.py
                            admin/admin_routes.py
                            product/product_routes.py
```

파일을 폴더로 더 정리한 것뿐. 개념은 똑같음

폴더 안으로 들어가면 import 경로만 바뀜

```python
# 1.basic (같은 폴더)
from user_routes import user_blueprint

# 2.more (폴더 안에 있음)
from routes.user.user_routes import user_blueprint
# routes 폴더 > user 폴더 > user_routes.py
```
