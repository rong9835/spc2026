# 게시판 미니앱 핵심 정리

> 처음 배우는 사람을 위한 흐름 + 패턴 요약

---

## 전체 구조 한눈에 보기

```
브라우저 (HTML + JS)
    ↕ fetch (HTTP 요청/응답)
Flask 서버 (app.py)
    ↕ SQL 쿼리
SQLite DB (board.sqlite)
```

**3계층 구조**가 핵심이에요.

| 계층 | 파일 | 역할 |
|------|------|------|
| 화면 (Frontend) | `static/index.html`, `static/js/script.js` | 사용자에게 보여주고, 입력받기 |
| 서버 (Backend) | `app.py` | 요청을 받아서 DB에 전달 |
| 데이터베이스 | `database.py`, `board.sqlite` | 데이터 영구 저장 |

---

## 만드는 순서 (6단계)

### 1단계 — DB 테이블 만들기 (`database.py`)

가장 먼저! 데이터를 **어디에 어떻게 저장할지** 정의합니다.

```python
CREATE TABLE IF NOT EXISTS board (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,  -- 자동 번호
    title   TEXT NOT NULL,                      -- 제목
    message TEXT NOT NULL                       -- 내용
)
```

> 비유: 엑셀 파일을 열고 첫 행에 **열 이름(제목, 내용)**을 먼저 적는 것

---

### 2단계 — 저장 API 만들기 (`/create`)

브라우저가 보낸 데이터를 DB에 **INSERT** 합니다.

```python
@app.route('/create', methods=['POST'])
def create():
    title   = request.json.get('title')    # 브라우저에서 받아오기
    message = request.json.get('message')
    sql = 'INSERT INTO board (title,message) VALUES (?,?)'
    db.execute(sql, (title, message))
    db.commit()
    return jsonify({'result': 'success'})  # 성공 응답 돌려주기
```

> `?` 는 자리표시자 — SQL 인젝션 방지용 (값을 직접 끼워넣지 않음)

---

### 3단계 — 목록 API 만들기 (`/list`)

DB에서 전체 데이터를 **SELECT** 해서 브라우저로 보냅니다.

```python
@app.route('/list')
def list():
    sql = 'SELECT * FROM board'
    result = db.execute_fetch(sql)
    return jsonify({'result': result})
```

---

### 4단계 — 버튼 클릭 → 서버로 전송 (`script.js`)

브라우저에서 **fetch**로 서버 API를 호출합니다.

```javascript
document.getElementById('submitBtn').addEventListener('click', function () {
    let title   = document.getElementById('input-title').value;
    let message = document.getElementById('input-text').value;

    fetch('/create', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ title: title, message: message }),
    }).then(response => response.json())
      .then(data => {
          loadList();  // 저장 후 목록 새로고침
      });
});
```

---

### 5단계 — 화면에 목록 표시

`/list` API를 호출해서 받은 데이터를 **HTML 카드로 변환**합니다.

```javascript
function loadList() {
    fetch('/list')
        .then(response => response.json())
        .then(data => {
            let cardList = document.getElementById('card-list');
            cardList.innerHTML = '';  // 기존 목록 초기화

            data.result.forEach(function (item) {
                let id      = item[0];
                let title   = item[1];
                let message = item[2];

                let card = document.createElement('div');
                card.innerHTML = `<h3>${title}</h3><p>${message}</p>`;
                cardList.appendChild(card);
            });
        });
}

loadList();  // 페이지 열자마자 목록 불러오기
```

---

### 6단계 — 삭제 / 수정 (확장)

같은 패턴으로 `/delete`, `/modify` API를 추가합니다.

```python
@app.route('/delete', methods=['POST'])
def delete():
    id = request.json.get('id')
    sql = 'DELETE FROM board WHERE id=?'
    db.execute(sql, (id,))
    db.commit()
    return jsonify({'result': 'success'})
```

## 핵심 패턴 3가지

### 패턴 1 — 요청-응답 사이클

```
브라우저                     Flask 서버
   │                             │
   │─── fetch('/create', POST) ──▶│
   │                             │── DB에 INSERT
   │◀── {'result': 'success'} ───│
   │                             │
```

1. 브라우저가 **요청(Request)** 보냄
2. 서버가 DB 작업 후 **응답(Response)** 보냄
3. 브라우저가 응답 받아 화면 업데이트

---

### 패턴 2 — CRUD

모든 게시판/앱의 기본 4가지 동작:

| 동작 | SQL | HTTP 메서드 | URL |
|------|-----|------------|-----|
| **C**reate (저장) | `INSERT` | `POST` | `/create` |
| **R**ead (조회) | `SELECT` | `GET` | `/list` |
| **U**pdate (수정) | `UPDATE` | `POST` | `/modify` |
| **D**elete (삭제) | `DELETE` | `POST` | `/delete` |

---

### 패턴 3 — DB 클래스 분리

`database.py`를 별도 파일로 만들어서 `app.py`가 DB 세부사항을 몰라도 되게 합니다.

```python
# app.py 에서는 이렇게만 쓰면 됨
db = MyDatabase()
db.execute(sql, args)      # 쓰기 (INSERT, UPDATE, DELETE)
db.execute_fetch(sql)      # 읽기 (SELECT)
db.commit()                # 저장 확정
```

> 비유: 편의점 직원이 창고 구조를 몰라도 "재고 가져와"라고 말하면 되는 것


## 파일별 한 줄 요약

| 파일 | 한 줄 요약 |
|------|-----------|
| `database.py` | DB 연결 + 테이블 생성 + SQL 실행 도구 모음 |
| `app.py` | URL별로 어떤 작업 할지 정의 (라우팅 + 비즈니스 로직) |
| `index.html` | 사용자 눈에 보이는 화면 (입력창, 버튼, 목록 영역) |
| `script.js` | 버튼 클릭 감지 → 서버 통신 → 화면 업데이트 |

## 자주 헷갈리는 것들

### Q. `jsonify`는 왜 써요?
Python 딕셔너리(`dict`)를 브라우저가 읽을 수 있는 **JSON 문자열**로 변환해줘요.

### Q. `commit()`은 왜 따로 호출해요?
DB 변경은 임시 저장 상태예요. `commit()`을 해야 **실제로 파일에 기록**됩니다.  
commit을 안 하면 서버 재시작 시 데이터가 날아가요.

### Q. `methods=['POST']`는 뭔가요?
해당 URL이 **어떤 HTTP 방식**을 받는지 명시해요.
- `GET` — 데이터 조회 (기본값, URL에 데이터 포함)
- `POST` — 데이터 전송 (body에 데이터 숨김)

### Q. `?` 자리표시자는 왜 써요?
```python
# 위험 (SQL 인젝션 가능)
sql = f"INSERT INTO board VALUES ('{title}')"

# 안전 (자리표시자 사용)
sql = "INSERT INTO board VALUES (?)"
db.execute(sql, (title,))
```
직접 문자열을 끼워넣으면 해커가 SQL을 조작할 수 있어요.
