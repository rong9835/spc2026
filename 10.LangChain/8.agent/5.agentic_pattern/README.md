## 1. Prompt Chaining (프롬프트 체이닝)

> **파일:** `1.prompt_chaining.py`

### 쉽게 말하면?

**공장 컨베이어 벨트**처럼 AI가 순서대로 일을 넘겨받아 처리하는 방식이에요.

```
[1단계 AI] → [2단계 AI] → [3단계 AI] → [4단계 AI]
  리서치   →  품질검사  →    분석     →   보고서
```

### 코드에서는 이렇게 했어요

| 단계 | 하는 일 | 비유 |
|------|---------|------|
| 1단계 | 주제에 대해 핵심 사실 5가지 조사 | 신입사원이 자료 수집 |
| 2단계 | 조사 결과가 괜찮은지 PASS/FAIL 판정 | 팀장이 검토 |
| 3단계 | 조사 결과를 바탕으로 심층 분석 | 전문가가 분석 |
| 4단계 | 최종 CEO 보고서 작성 | 임원 보고용 문서 완성 |

### 핵심 포인트: 게이트(Gate)

2단계에서 **FAIL이 나오면 다시 합니다!**

```python
if gate_result.lower() in "fail":
    print("게이트 검증에 실패하여 해당 업무를 재수행 합니다.")
    gate_result = gate_chain.invoke({'research': research})
```

> 마치 공장에서 불량품이 나오면 **다시 만드는 것**처럼, AI도 품질이 안 좋으면 재작업해요.

### 왜 쓰나요?

- 한 번에 복잡한 일을 시키면 AI가 엉터리로 할 수 있어요
- **단계를 나누면** 각 단계에서 잘 할 수 있는 일만 집중해서 함
- 중간에 검사(게이트)를 넣어서 **품질을 보장**할 수 있어요

## 2. Routing (라우팅)

> **파일:** `2.routing.py`

### 쉽게 말하면?

**고객센터 안내 전화**처럼, 문의 내용에 따라 **담당 부서로 연결**해주는 방식이에요.

```
고객 문의
   ↓
[분류 AI] → "기술 문의네?" → [기술 지원 AI]
          → "결제 문의네?" → [결제 담당 AI]
          → "일반 문의네?" → [일반 상담 AI]
```

### 코드에서는 이렇게 했어요

**3가지 전문 챗봇**을 만들었어요:

| 챗봇 | 역할 | 예시 질문 |
|------|------|---------|
| `technical_chain` | 기술 문제 해결 | "프로그램이 충돌해요" |
| `billing_chain` | 결제/환불 처리 | "환불하고 싶어요" |
| `general_chain` | 일반 상담 | "어떤 기능이 있나요?" |

**분류 AI**가 먼저 어느 쪽으로 보낼지 결정:

```python
category = classifier_chain.invoke({"question": question})
# → "technical" or "billing" or "general"

chain = route_map.get(category, general_chain)
# → 해당 카테고리의 챗봇에 연결!
```

### 실제 테스트 예시

```
질문: "프로그램이 자꾸 충돌하는데 어떻게 해야 하나요?"
→ 분류: technical
→ 기술 지원 전문가 AI가 답변

질문: "구독을 취소하고 환불받고 싶습니다."
→ 분류: billing
→ 결제 담당 AI가 답변
```

### 왜 쓰나요?

- 모든 질문을 하나의 AI가 처리하면 **전문성이 떨어져요**
- **전문가별로 나누면** 각 분야에 최적화된 답변 가능
- 마치 병원에서 **내과/외과/소아과**로 나눈 것과 같은 원리!

## 3. Parallel (병렬 처리)

> **파일:** `3.parallel.py`

### 쉽게 말하면?

**여러 심사위원이 동시에 채점**하는 방식이에요. 한 명씩 순서대로 하면 오래 걸리지만, 동시에 하면 빠르죠!

```
         ┌→ [심사위원 1 (냉정하게)] ─┐
번역문 ──┼→ [심사위원 2 (보통으로)] ─┼→ 최종 점수
         └→ [심사위원 3 (창의적으로)] ─┘
```

### 코드에서는 이렇게 했어요

**같은 번역을 3명의 AI가 동시에 평가**해요:

```python
llm1 = ChatOpenAI(temperature=0.0)  # 냉정하고 일관된 평가
llm2 = ChatOpenAI(temperature=0.5)  # 균형잡힌 평가
llm3 = ChatOpenAI(temperature=1.0)  # 창의적이고 다양한 평가

voter1 = vote_prompt | llm1 | parser
voter2 = vote_prompt | llm2 | parser
voter3 = vote_prompt | llm3 | parser
```

> **temperature**란? AI의 "창의성 레벨"이에요  
> - `0.0` = 항상 같은 답 (로봇처럼 일관됨)  
> - `1.0` = 매번 다른 답 (창의적이고 다양함)

### RunnableParallel이란?

```python
parallel_vote = RunnableParallel(
    voter1 = voter1,
    voter2 = voter2,
    voter3 = voter3,
)
# → 3개를 동시에 실행! 시간이 1/3로 줄어요
```

### 왜 쓰나요?

- 하나의 AI 판단은 편향될 수 있어요 → **여러 관점에서 평가**
- 순서대로 하면 3배 시간이 걸리지만, 동시에 하면 **1배 시간**
- 마치 **시험 채점을 3명이 동시에** 하는 것과 같아요

## 세 가지 패턴 비교

| 패턴 | 핵심 아이디어 | 비유 | 언제 쓰나? |
|------|------------|------|----------|
| **Chaining** | 순서대로 단계별 처리 | 컨베이어 벨트 | 복잡한 작업을 단계로 나눌 때 |
| **Routing** | 내용에 따라 다른 곳으로 | 안내 데스크 | 질문 종류가 여러 가지일 때 |
| **Parallel** | 여러 AI가 동시에 처리 | 동시 채점 | 같은 일을 빠르게 여러 관점으로 |

---

## 전체 흐름 한눈에 보기

```
사용자 입력
    │
    ├── 복잡한 작업? → Chaining (단계별 처리)
    │
    ├── 종류가 다양? → Routing (담당자 연결)
    │
    └── 빠른 다중 평가? → Parallel (동시 처리)
```

> **핵심 요약:**  
> AI를 한 번만 쓰는 게 아니라, **여러 AI를 조합해서 팀처럼 일하게 만드는 것**이 에이전틱 패턴이에요! 🤝
