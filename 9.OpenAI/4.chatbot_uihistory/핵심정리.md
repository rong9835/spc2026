> 이전 실습(3번)과 비교해서 **딱 한 가지**가 추가됐어요.  
> 바로 **"대화 기억"** 기능입니다!

---

## 이전 챗봇 vs 이번 챗봇 차이점

| 구분 | 이전 챗봇 (3번) | 이번 챗봇 (4번) |
|------|----------------|----------------|
| 기억 | 각 대화가 독립적 (기억 못함) | 대화 전체를 기억함 |
| 예시 | "내 이름은 홍길동이야" → 다음 질문에서 이름 모름 | "내 이름은 홍길동이야" → 다음 질문에서 이름 기억! |

---

## 핵심 개념: 히스토리(history)란?

```python
history = []  # 대화 기록을 담는 리스트 (빈 박스로 시작)
```

**비유로 이해하기:**  
카카오톡 채팅방과 똑같아요!  
대화할수록 메시지가 쌓이고, GPT는 그 전체 대화를 보고 답변합니다.

```
history 안에 쌓이는 모습:
[
  { role: 'user',      content: '내 이름은 홍길동이야' },
  { role: 'assistant', content: '안녕하세요 홍길동님!' },
  { role: 'user',      content: '내 이름이 뭐야?' },       ← 새 질문
  { role: 'assistant', content: '홍길동이라고 하셨잖아요!' } ← GPT 답변
]
```

---

## 전체 흐름 이해하기

```
사용자 입력
    ↓
history에 추가 (role: 'user')
    ↓
GPT에게 [시스템 설정 + 전체 history] 전달
    ↓
GPT 답변 받기
    ↓
history에 추가 (role: 'assistant')
    ↓
화면에 표시
```

### 2. /api/chat 라우트 - 대화 기록 쌓기

```python
@app.route('/api/chat', methods=['POST'])
def chat():
    data = request.get_json()
    chat_message = data.get('chatMessage', '')

    # ① 사용자 메시지를 history에 추가
    history.append({'role': 'user', 'content': chat_message})

    # ② GPT에게 물어보기
    gpt_reply = ask_chatgpt(chat_message)

    # ③ GPT 답변도 history에 추가
    history.append({'role': 'assistant', 'content': gpt_reply})

    return jsonify({'reply': gpt_reply})
```

---

### 3. ask_chatgpt 함수 - 전체 대화를 GPT에게 전달

```python
def ask_chatgpt(chat_message):
    gpt_ask_message = [
        {"role": "system", "content": "당신은 친절한 챗봇입니다. 경상도 사투리를 섞어서 답변하시오."},
        *history   # ← 이게 핵심! history를 펼쳐서 통째로 전달
    ]
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=gpt_ask_message
    )
    return response.choices[0].message.content
```

#### `*history` 란?

```python
# *history = 리스트를 펼치기 (스프레드)

history = [{'role': 'user', 'content': '안녕'}]

# *history 를 사용하면:
[system_message, {'role': 'user', 'content': '안녕'}]
# 이렇게 system 뒤에 대화가 쭉 붙어요
```

## GPT에게 전달되는 최종 메시지 구조

```
[
  { role: "system",    content: "친절한 챗봇, 경상도 사투리" },  ← 항상 고정
  { role: "user",      content: "내 이름은 홍길동이야" },        ← 첫 번째 대화
  { role: "assistant", content: "안녕하세요 홍길동님!" },
  { role: "user",      content: "내 이름이 뭐야?" }             ← 지금 질문
]
```

GPT는 위 대화 전체를 읽고 답변하기 때문에 **이전 내용을 기억**할 수 있어요.

## 프론트엔드 (script.js) 핵심 구조

JS 코드는 **3가지 역할로 분리**되어 있어요:

```
① 이벤트 감지    →  formInput.addEventListener('submit', ...)
                     폼 제출 시 실행, 입력값 가져오기

② API 통신       →  fetchChatbotReply(message)
                     서버(/api/chat)에 POST 요청, 답변 받기

③ 화면 그리기    →  appendMessage(sender, text)
                     말풍선 HTML 동적으로 생성해서 추가
```

### 메시지 화면에 추가하는 함수

```javascript
function appendMessage(sender, text) {
    const messageDiv = document.createElement('div');
    messageDiv.classList.add('message', sender); // 'user' or 'bot'

    const bubbleDiv = document.createElement('div');
    bubbleDiv.classList.add('bubble');
    bubbleDiv.innerText = text;

    messageDiv.appendChild(bubbleDiv);
    resultDiv.appendChild(messageDiv);

    // 자동 스크롤 (항상 최신 메시지가 보이도록)
    chatContainer.scrollTop = chatContainer.scrollHeight;
}
```

> `sender`가 `'user'`면 파란 말풍선 (우측), `'bot'`이면 회색 말풍선 (좌측)으로 CSS가 자동 적용돼요.

## role의 3가지 종류

| role | 역할 | 설명 |
|------|------|------|
| `system` | 시스템 | AI의 성격/규칙 설정. 매번 맨 앞에 붙음 |
| `user` | 사용자 | 내가 보낸 메시지 |
| `assistant` | AI | GPT가 보낸 답변 |

## 이번 실습의 한계점

| 문제 | 이유 | 해결책 |
|------|------|--------|
| 서버 재시작 시 대화 사라짐 | `history`가 메모리에만 저장 | 5번 실습 - SQLite DB 사용 |

| 대화가 길어질수록 API 비용 증가 | 매번 전체 history를 전송 | 최근 N개만 보내도록 제한 |
