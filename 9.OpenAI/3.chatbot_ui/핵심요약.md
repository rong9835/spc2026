# 챗봇 UI 만들기 핵심요약

## 프로젝트 구조

```
3.chatbot_ui/
├── app.py              ← 서버 (Flask, Python)
└── static/
    ├── index.html      ← 화면 뼈대
    ├── style.css       ← 화면 꾸미기
    └── script.js       ← 화면 동작
```

> **비유:** 레스토랑으로 생각하면 — `index.html`은 식당 인테리어, `style.css`는 데코레이션, `script.js`는 직원(서빙), `app.py`는 주방(요리사)

---

## 전체 흐름 한눈에 보기

```
사용자 입력
    ↓
[브라우저 - script.js]
    ↓ fetch('/api/chat') POST 요청
[서버 - app.py]
    ↓ client.chat.completions.create()
[OpenAI GPT API]
    ↑ 답변 반환
[서버 - app.py]
    ↑ JSON으로 응답 { reply: "..." }
[브라우저 - script.js]
    ↑ 말풍선으로 화면에 출력
```

---

## 핵심 1 — 서버 (app.py)

### Flask 앱 설정

```python
app = Flask(__name__, static_folder='static', static_url_path='')
```

- `static_folder='static'` → HTML/CSS/JS 파일이 있는 폴더 지정
- `static_url_path=''` → URL에서 `/static/` 없이 바로 접근 가능하게 함
  - 예: `http://localhost:5000/style.css` (원래는 `/static/style.css`)

### 라우트 2개

| 라우트 | 역할 |
|--------|------|
| `GET /` | `index.html` 화면 보내주기 |
| `POST /api/chat` | 메시지 받아서 GPT에 물어보고 답변 반환 |

### ChatGPT 호출 핵심

```python
messages = [
    {"role": "system", "content": "당신은 친절한 챗봇입니다. 경상도 사투리를 섞어서 답변하시오."},
    {"role": "user",   "content": chat_message}
]
```

- **system** = 챗봇의 성격/역할을 정해주는 지시문
- **user** = 실제 사용자가 보낸 메시지
- 답변은 `response.choices[0].message.content`에 들어있음

## 핵심 2 — 화면 동작 (script.js)

### 역할 분리 (함수를 2개로 쪼갬)

```
폼 제출 이벤트
    ├── fetchChatbotReply()  ← API 통신만 담당
    └── appendMessage()      ← 화면 그리기만 담당
```

> 하나의 함수가 모든 걸 하면 코드가 복잡해짐. 역할을 나눠야 읽기 쉽고 수정하기 쉬움.

### fetch 통신 (서버에 요청 보내기)

```javascript
const response = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ chatMessage: message })
});
const data = await response.json();
return data.reply;
```

- `fetch` = 브라우저가 서버에 HTTP 요청을 보내는 함수
- `await` = 응답이 올 때까지 기다림 (비동기 처리)
- `JSON.stringify()` = JS 객체 → JSON 문자열로 변환 (전송용)
- `response.json()` = 받은 JSON 문자열 → JS 객체로 변환

### 말풍선 만들기 (DOM 조작)

```javascript
function appendMessage(sender, text) {
    const messageDiv = document.createElement('div');
    messageDiv.classList.add('message', sender);  // 'message user' or 'message bot'

    const bubbleDiv = document.createElement('div');
    bubbleDiv.classList.add('bubble');
    bubbleDiv.innerText = text;

    messageDiv.appendChild(bubbleDiv);
    resultDiv.appendChild(messageDiv);
}
```

- `createElement` = HTML 태그를 JS로 만드는 함수
- `classList.add` = 클래스 추가 (CSS 스타일 적용용)
- `appendChild` = 부모 안에 자식 태그 추가

## 핵심 3 — 화면 꾸미기 (style.css)

### 말풍선 좌우 정렬 원리

```
사용자 메시지 .message.user  → justify-content: flex-end   (오른쪽 정렬)
챗봇 메시지   .message.bot   → justify-content: flex-start (왼쪽 정렬)
```

```css
/* 사용자: 파란색 말풍선, 오른쪽 */
.message.user { justify-content: flex-end; }
.message.user .bubble { background-color: #007bff; color: white; }

/* 챗봇: 회색 말풍선, 왼쪽 */
.message.bot { justify-content: flex-start; }
.message.bot .bubble { background-color: #e9ecef; color: #333; }
```

### 레이아웃 구조 (Flexbox)

```
chatbot-wrapper (flex, column 방향)
├── h1 (제목)
├── chat-container (flex: 1 → 남은 공간 다 차지, 스크롤)
│   └── result (메시지들이 쌓이는 곳)
└── user-input-form (입력창 + 버튼)
```

- `flex: 1` = 가능한 공간을 모두 차지하라는 의미 → 대화창이 늘어남
- `overflow-y: auto` = 내용이 넘치면 스크롤 생김

## 자주 헷갈리는 포인트

| 헷갈리는 것 | 설명 |
|-------------|------|
| `ev.preventDefault()` | 폼 기본 동작(페이지 새로고침)을 막음 |
| `async / await` | 서버 응답을 기다려야 하니까 비동기로 처리 |
| `chatContainer.scrollTop = chatContainer.scrollHeight` | 새 메시지 올 때마다 자동으로 스크롤 맨 아래로 이동 |
| `send_from_directory` | Flask에서 정적 파일(HTML)을 브라우저에 보내주는 함수 |
| `jsonify()` | Python dict → JSON 응답으로 변환하는 Flask 함수 |
