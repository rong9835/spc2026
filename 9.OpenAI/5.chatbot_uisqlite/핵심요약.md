## SQLite 핵심 개념 3가지

### 1. 연결 (Connect) — DB 파일 열기
```python
conn = sqlite3.connect("chatgpt.db", check_same_thread=False)
```
- `chatgpt.db` 파일이 없으면 자동으로 만들어줌
- `check_same_thread=False` → Flask 여러 요청에서 같은 연결 쓸 수 있게 허용

### 2. 커서 (Cursor) — SQL 명령을 실행하는 도구
```python
cursor = conn.cursor()
```
- 실제로 SQL 명령을 실행하는 객체
- 비유: 연결(`conn`)은 **데이터베이스 창구**, 커서(`cursor`)는 **창구 직원**

### 3. 커밋 (Commit) — 변경사항 저장 확정
```python
conn.commit()
```
- INSERT/UPDATE/DELETE 후 반드시 호출해야 실제로 저장됨
- 비유: 문서 작성 후 **저장(Ctrl+S)** 누르는 것

## 테이블 구조

```sql
CREATE TABLE IF NOT EXISTS history (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,  -- 자동 증가 번호
    role      TEXT NOT NULL,                       -- 'user' 또는 'assistant'
    content   TEXT NOT NULL,                       -- 메시지 내용
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP  -- 저장 시각 (자동)
)
```

| 컬럼 | 의미 | 예시 |
|------|------|------|
| `id` | 자동으로 붙는 번호 | 1, 2, 3 ... |
| `role` | 누가 보낸 메시지? | `"user"` / `"assistant"` |
| `content` | 메시지 내용 | `"안녕하세요"` |
| `timestamp` | 언제 보냈는지 | `2026-05-22 10:30:00` |

## 핵심 코드 분석 (app.py)

### 메시지 저장 (INSERT)
```python
# 이전: history.append({'role': 'user', 'content': chat_message})
cursor.execute("INSERT INTO history (role, content) VALUES (?, ?)", ('user', chat_message))
conn.commit()
```
- `?` → SQL 인젝션 방지를 위한 안전한 값 입력 방식

### 대화 기록 불러오기 (SELECT)
```python
cursor.execute("SELECT role, content FROM history ORDER BY id DESC LIMIT 10")
rows = cursor.fetchall()
rows = rows[::-1]  # 최신순 → 오래된순으로 뒤집기
```

**왜 DESC로 가져와서 뒤집는가?**
```
DB에서 가져올 때:  [10번, 9번, 8번, 7번, ...]  (최신이 먼저)
뒤집은 후:        [7번, 8번, 9번, 10번, ...]   (오래된 게 먼저)
GPT에게 전달:      시간 순서대로 대화 흐름 파악 가능 ✅
```

### Row를 딕셔너리로 변환
```python
conn.row_factory = sqlite3.Row  # 이걸 설정해야

rows_dict = [{'role': row['role'], 'content': row['content']} for row in rows]
```
- `sqlite3.Row` 설정 덕분에 `row['role']` 처럼 컬럼 이름으로 접근 가능

### GPT에게 보낼 최종 메시지 조립
```python
gpt_ask_message = [
    {"role": "system", "content": "당신은 친절한 챗봇입니다. 경상도 사투리를 적절하게 섞어서 답변하시오."},
    *rows_dict   # DB에서 불러온 대화 기록을 펼쳐서 넣기
]
```
- `*rows_dict` → 리스트를 펼쳐서 하나씩 넣는 문법 (스프레드 연산자)

## SQL 명령어 4가지 요약

| 명령어 | 용도 | 이번 실습에서 사용 |
|--------|------|------------------|
| `CREATE TABLE` | 테이블 만들기 | ✅ `init_db()` 함수 |
| `INSERT INTO` | 데이터 넣기 | ✅ 메시지 저장 |
| `SELECT` | 데이터 읽기 | ✅ 대화 기록 불러오기 |
| `UPDATE / DELETE` | 수정/삭제 | ❌ (이번엔 미사용) |

## 데이터베이스를 써야 하는 이유

```
❌ 리스트(history = [])의 문제점
  - 서버를 껐다 켜면 기록이 전부 사라짐
  - 여러 사용자가 동시에 쓰면 기록이 섞임
  - 메모리 용량 한계가 있음

✅ SQLite의 장점
  - 파일로 저장되어 서버 재시작 후에도 유지
  - SQL로 원하는 데이터만 쏙 뽑아오기 가능
  - 용량이 커져도 효율적으로 관리 가능
```

## 자주 헷갈리는 포인트

### Q. `check_same_thread=False`는 왜 필요한가?
Flask는 여러 사용자 요청을 동시에 처리하는데, SQLite는 기본적으로
하나의 스레드에서만 사용하도록 설정되어 있어요.
이 옵션을 끄면 여러 스레드에서 같은 DB 연결을 쓸 수 있습니다.

### Q. `conn.row_factory = sqlite3.Row`는 왜 쓰는가?
이 설정 없이는 `rows[0][0]` 처럼 숫자로만 접근 가능해요.
설정 후에는 `row['role']` 처럼 컬럼 이름으로 접근 가능해 코드가 읽기 쉬워집니다.

### Q. `LIMIT 10`은 왜 10개만 가져오는가?
GPT API는 보내는 메시지가 많을수록 비용이 비싸지기 때문에,
최근 대화 10개만 맥락으로 전달해서 비용을 절약합니다.
