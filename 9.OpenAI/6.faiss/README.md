# FAISS + RAG 핵심 요약

> **초보자를 위한 쉬운 설명** — "내 데이터로 GPT에게 답변시키기"

---

## 💡 RAG가 뭔가요?

**RAG = Retrieval-Augmented Generation** (검색 + 생성)

### 문제 상황
GPT는 모든 걸 알고 있는 것처럼 보이지만, **내 회사 자료 / 내 개인 데이터**는 모릅니다.

```
❌ 일반 GPT: "홍길동이 누구야?" → "모르겠어요"
✅ RAG GPT:  "홍길동이 누구야?" → 내 문서에서 찾아서 → "강원도 출신이고..."
```

### 도서관 사서 비유
> 📚 RAG는 **도서관 사서**와 같습니다.
> 
> 1. 사용자가 질문을 합니다
> 2. 사서(RAG)가 도서관(내 데이터)에서 관련 책을 찾습니다
> 3. 그 책의 내용을 GPT에게 보여주면서 답변을 요청합니다
> 4. GPT가 그 책 내용을 바탕으로 답변합니다

---

## 🔢 임베딩(Embedding)이 뭔가요?

**텍스트를 숫자(벡터)로 바꾸는 과정**

### 왜 숫자로 바꾸나요?
컴퓨터는 글자의 "의미"를 직접 비교할 수 없습니다.  
숫자로 바꾸면 **얼마나 비슷한 의미인지** 계산할 수 있어요!

```
"강아지" → [0.2, 0.8, 0.1, 0.5, ...]  (1536개의 숫자)
"개"     → [0.2, 0.7, 0.1, 0.4, ...]  (비슷한 숫자!)
"자동차" → [0.9, 0.1, 0.8, 0.2, ...]  (전혀 다른 숫자)
```

> 📏 비유: 지도에서 위치를 좌표(숫자)로 나타내면  
> 두 도시가 얼마나 가까운지 계산할 수 있는 것과 같습니다!

OpenAI의 `text-embedding-ada-002` 모델은 텍스트를 **1536개의 숫자**로 변환합니다.

---

## 🗄️ FAISS가 뭔가요?

**Facebook이 만든 초고속 숫자 검색 도구**

수천~수백만 개의 벡터(숫자 묶음) 중에서  
**가장 비슷한 것을 빠르게 찾아주는** 데이터베이스입니다.

```python
index = faiss.IndexFlatL2(1536)  # 1536차원 벡터를 저장하는 인덱스 생성
index.add(doc_embeddings)        # 문서 벡터들을 저장
index.search(query_embedding, k=1)  # 가장 비슷한 문서 1개 찾기
```

> 🔍 비유: **천만 장의 사진** 중에서 내 얼굴과 가장 닮은 사진을  
> 1초 안에 찾아주는 것과 같습니다!

## 🏗️ 전체 동작 흐름

```
[1단계] 데이터 준비
  내 문서들 → OpenAI 임베딩 → 숫자 벡터로 변환 → FAISS에 저장

[2단계] 질문 처리
  사용자 질문 → OpenAI 임베딩 → 숫자 벡터로 변환

[3단계] 유사 문서 검색
  FAISS에서 "질문 벡터"와 가장 가까운 "문서 벡터" 찾기

[4단계] GPT에게 전달
  "이 자료를 바탕으로 질문에 답해줘" → GPT 응답
```

## 📄 파일 설명

### `1.intro_rag.py` — 기본 RAG

**핵심 코드 설명:**

```python
# 1. 나만의 데이터 준비
documents = [
    "한국소프트웨어저작권협회는 SPC라는 약자...",
    "홍길동은 2020년 1월 1일 생으로...",
    "Python 은 개발 언어중에 가장 쉽다고..."
]

# 2. 모든 문서를 숫자(벡터)로 변환해서 FAISS에 저장
index = faiss.IndexFlatL2(1536)          # 벡터 DB 생성
doc_embeddings = [get_embedding(doc) for doc in documents]  # 숫자로 변환
index.add(doc_embeddings)                # DB에 저장

# 3. 질문이 들어오면 → 가장 비슷한 문서 찾기 → GPT에게 전달
_, indices = index.search(query_embedding, k=1)   # 가장 가까운 문서 1개
retrieved_doc = documents[indices[0][0]]           # 그 문서 꺼내기
```

**문제점:** 질문과 전혀 관계없는 질문("오늘 저녁 뭐 먹을까?")도  
억지로 문서를 찾아서 엉뚱한 답변을 할 수 있음

---

### `2.better_rag.py` — 개선된 RAG (유사도 필터링 추가)

**추가된 핵심 코드:**

```python
# 거리값을 유사도 점수(0~1)로 변환
distance, indices = index.search(np.array([query_embedding]), k=1)
true_distance = np.sqrt(distance[0][0])
similarity_score = 1 / (1 + true_distance)  # 1에 가까울수록 유사함

# 유사도가 낮으면 → 답변 거부!
if similarity_score < 0.65:
    return "해당 내용은 적합한 답변을 찾을 수 없습니다."
```

**유사도 점수 해석:**
| 점수 | 의미 |
|------|------|
| 0.9 이상 | 매우 관련 있음 |
| 0.65 ~ 0.9 | 관련 있음 → 답변 |
| 0.65 미만 | 관련 없음 → 답변 거부 |

## 🔄 1 vs 2 비교

| 구분 | `1.intro_rag.py` | `2.better_rag.py` |
|------|-----------------|-------------------|
| 유사도 필터링 | ❌ 없음 | ✅ 있음 (0.65 기준) |
| 관련 없는 질문 | 억지로 답변 | 정중히 거부 |
| 신뢰도 | 낮음 | 높음 |
| 코드 복잡도 | 단순 | 약간 복잡 |

## 🧩 전체 구조 한눈에 보기

```
사용자 질문
    │
    ▼
[get_embedding()]         ← 질문을 숫자로 변환 (OpenAI API)
    │
    ▼
[FAISS index.search()]    ← 내 데이터에서 가장 비슷한 문서 찾기
    │
    ▼
[유사도 점수 확인]         ← 0.65 미만이면 답변 거부 (2번 파일만)
    │
    ▼
[GPT에게 질문 + 문서 전달] ← "이 자료 보고 답해줘"
    │
    ▼
최종 답변 출력
```

## 💬 핵심 용어 정리

| 용어 | 쉬운 설명 |
|------|-----------|
| **임베딩(Embedding)** | 텍스트를 숫자 목록으로 변환하는 것 |
| **벡터(Vector)** | 숫자들의 목록 `[0.1, 0.5, 0.3, ...]` |
| **FAISS** | 숫자 목록들 사이의 거리를 빠르게 계산하는 도구 |
| **L2 거리** | 두 점 사이의 직선 거리 (숫자가 작을수록 유사) |
| **RAG** | 내 데이터에서 관련 내용을 찾아 GPT에게 제공하는 방식 |
| **컨텍스트** | GPT에게 같이 보내주는 참고 자료 |
