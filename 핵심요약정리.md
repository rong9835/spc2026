## 1. HTML — 웹페이지의 뼈대

> **비유:** HTML = 건물의 뼈대 (기둥, 벽, 문)

```html
<!DOCTYPE html>
<html>
  <head>
    <title>페이지 제목</title>
  </head>
  <body>
    <h1>큰 제목</h1>
    <p>단락 텍스트</p>
    <a href="링크주소">클릭하면 이동</a>
    <img src="이미지경로" alt="설명">
    <button>클릭 버튼</button>
  </body>
</html>
```

### 자주 쓰는 태그 정리

| 태그 | 역할 | 예시 |
|------|------|------|
| `<h1>~<h6>` | 제목 (1이 가장 큼) | `<h1>제목</h1>` |
| `<p>` | 문단 텍스트 | `<p>내용</p>` |
| `<div>` | 구역 나누기 (박스) | `<div>...</div>` |
| `<span>` | 글자 일부 꾸미기 | `<span>일부</span>` |
| `<a>` | 링크 | `<a href="url">클릭</a>` |
| `<img>` | 이미지 | `<img src="파일.jpg">` |
| `<ul>/<li>` | 목록 | `<ul><li>항목</li></ul>` |
| `<form>` | 입력 폼 | `<form>...</form>` |
| `<input>` | 입력창 | `<input type="text">` |

## 7. Flask — 파이썬 웹 서버

> **비유:** Flask = 음식을 요청받고 만들어서 내주는 주방

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

# GET 요청 처리 (데이터 주기)
@app.route('/api/users', methods=['GET'])
def get_users():
    users = [{"id": 1, "name": "홍길동"}]
    return jsonify(users)

# POST 요청 처리 (데이터 받기)
@app.route('/api/users', methods=['POST'])
def create_user():
    data = request.json          # 클라이언트가 보낸 데이터
    name = data['name']
    return jsonify({"message": "생성됨", "name": name}), 201

if __name__ == '__main__':
    app.run(debug=True)
```

### role 3가지

| role | 역할 | 비유 |
|------|------|------|
| `system` | AI 성격/지침 설정 | 직원 교육 매뉴얼 |
| `user` | 내가 하는 말 | 고객 질문 |
| `assistant` | AI가 한 말 | 직원 답변 |

### 대화 히스토리 (기억하는 챗봇)

```python
messages = []  # 대화 기록 바구니

def chat(user_input):
    messages.append({"role": "user", "content": user_input})
    
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages      # 전체 기록을 매번 보냄
    )
    
    answer = response.choices[0].message.content
    messages.append({"role": "assistant", "content": answer})
    return answer
```

> **핵심:** AI는 기억이 없다! 이전 대화를 매번 같이 보내줘야 기억하는 척 할 수 있다.

## 9. LangChain — AI 파이프라인 프레임워크

> **비유:** AI를 레고 블록처럼 조합해서 복잡한 앱을 만드는 도구

### 기본 구조 (LCEL)

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

llm = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_messages([
    ("system", "당신은 {role} 전문가입니다."),
    ("human", "{question}")
])
parser = StrOutputParser()

# 파이프(|)로 연결 = chain
chain = prompt | llm | parser

result = chain.invoke({"role": "요리사", "question": "파스타 레시피 알려줘"})
```

### 프롬프트 템플릿 종류

```python
# 2. 채팅 템플릿 (역할 지정)
```

### Chain 종류

```python
from langchain_core.runnables import RunnableLambda, RunnableParallel, RunnableBranch

# 1. 기본 체인 (순서대로)

# 2. 람다 함수 체인 (중간 변환)

# 3. 병렬 체인 (동시 실행)
parallel = RunnableParallel(

branch = RunnableBranch(

    (lambda x: "요리" in x["topic"], food_chain),
    general_chain  # 기본값
```

### RAG (LangChain 버전)

```python
from langchain_community.document_loaders import TextLoader, PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

# 1. 문서 로드
loader = TextLoader("문서.txt", encoding="utf-8")
docs = loader.load()

# 2. 청킹 (잘게 자르기)
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 3. 벡터 DB에 저장
vectorstore = Chroma.from_documents(chunks, OpenAIEmbeddings())

# 4. 검색 + 답변
retriever = vectorstore.as_retriever()
# context = retriever.invoke("질문") → 관련 문서 조각 반환
```

### Agent — AI가 도구를 선택해서 사용

```python
from langchain.agents import create_react_agent, AgentExecutor
from langchain_community.tools import WikipediaQueryRun

# 도구 정의
tools = [WikipediaQueryRun()]

# 에이전트 생성
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 실행 → AI가 스스로 도구를 선택해서 사용
executor.invoke({"input": "아인슈타인에 대해 알려줘"})
```

### 커스텀 도구 만들기

```python
from langchain.tools import tool

@tool
def get_weather(city: str) -> str:
    """도시의 날씨를 가져옵니다. city는 도시 이름입니다."""
    return f"{city}의 날씨는 맑음입니다."

# tools 리스트에 추가하면 AI가 필요할 때 알아서 호출
```

### 에이전틱 패턴 3가지

```
1. Chaining (체이닝) — 순서대로 단계별 처리
   [리서치 AI] → [검증 AI] → [분석 AI] → [보고서 AI]
   비유: 공장 컨베이어 벨트

2. Routing (라우팅) — 내용에 따라 다른 AI로 연결
   질문 → [분류 AI] → 기술 질문이면 [기술 AI]
                     → 결제 질문이면 [결제 AI]
   비유: 고객센터 안내 전화

3. Parallel (병렬) — 여러 AI가 동시에 처리
   번역문 → [심사 AI 1] ↘
          → [심사 AI 2] → [최종 판정]
          → [심사 AI 3] ↗
   비유: 여러 심사위원이 동시에 채점
```

### LangGraph — 상태 관리 그래프 (최신)

```python
from langgraph.graph import StateGraph, START, END, MessagesState

# 노드 = 하나의 처리 단계
def chatbot_node(state):
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

# 그래프로 연결
graph = StateGraph(MessagesState)
graph.add_node("chatbot", chatbot_node)
graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)

app = graph.compile()
```

> **LangGraph vs 일반 체인:**
> - 체인: 항상 같은 순서로만 실행
> - LangGraph: 조건에 따라 다른 경로 선택 가능, 반복/루프 가능

### 스트리밍 응답 패턴 (SSE)

```python
# 백엔드: 한 글자씩 스트리밍
from flask import Response, stream_with_context

@app.route('/api/stream')
def stream():
    def generate():
        for chunk in client.chat.completions.stream(...):
            text = chunk.choices[0].delta.content or ""
            yield f"data: {text}\n\n"
    return Response(stream_with_context(generate()), mimetype='text/event-stream')
```

## 14. 자주 헷갈리는 개념 정리

### AI 관련

| 개념 | 설명 | 비유 |
|------|------|------|
| **토큰** | AI가 읽고 쓰는 단위 | 영어는 단어, 한국어는 글자 |
| **temperature** | 창의성 조절 (0~1) | 0=항상 같은 답, 1=매번 다른 답 |
| **max_tokens** | 최대 답변 길이 | 비용 절감, 너무 긴 답변 방지 |
| **embedding** | 텍스트를 숫자 벡터로 변환 | 문장을 지도 위 좌표로 표현 |
| **RAG** | 내 문서를 참고해서 답변 | 오픈북 시험 |
| **Agent** | AI가 도구를 선택해서 사용 | 인턴이 필요한 것을 알아서 찾음 |
| **streaming** | 답변을 실시간으로 출력 | 타이핑하는 것처럼 보임 |

### 웹 관련

| 개념 | 설명 | 비유 |
|------|------|------|
| **API** | 기능을 외부에서 쓸 수 있게 제공 | 식당 주문 창구 |
| **REST API** | HTTP로 데이터를 주고받는 규칙 | 음식 주문 규칙 |
| **JSON** | 데이터를 주고받는 형식 | 표준 형식의 양식지 |
| **쿠키** | 브라우저에 저장되는 작은 데이터 | 방문 도장 |
| **세션** | 서버에서 사용자 정보 관리 | 음식점 번호표 |
| **CORS** | 다른 주소에서 API 호출 허용 | 외부 손님 입장 허가 |
